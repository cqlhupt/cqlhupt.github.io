Java JNI初探

Java作为一门高级开发语言，在我们工作生活中应用相当广泛，主要有Web开发，游戏开发，移动应用开发等。许多初学者可能会有一个错误的印象，认为Java是一门编译型语言，实际上Java的编译并不像C/C++的编译那样编译后能够直接得到可执行文件，Java以及一众基于JVM的语言（Kotlin、Scala等）编译后都只能得到字节码文件，运行时由JVM或者其他虚拟机解释执行。我们从字节码反编译得到Java的代码也相对简单一些。最终我们如果进入Java的底层，就不可避免的遇到JNI，让我们可以混合Java和C++进行开发，从而提高应用的性能。另一方面，了解JNI和JVM也能让我们更深入的了解Java的运行机制。

JVM和JNI

JVM（Java Virtual Machine）是Java虚拟机，负责解释运行字节码，垃圾回收，依赖加载等工作。JVM是由C/C++开发的，后续可能会出现其他语言开发的，比如Google现在已经为Android打造了Rust版本的ART（Android Runtime Environment）虚拟机。

JNI（Java Native Interface）是Java本地接口，它主要承担的是Java和C/C++语言交互的工作，我们可以将应用中需要高性能的部分使用C/C++开发，其他部分则使用Java开发。

问题汇总

  1. 在Windows上不能使用power shell和cmd进行gcc编译，具体原因不太清楚，可以使用git bash调用MinGW或者使用cgyWin
  2. gcc编译时必须指定三个头文件目录，否则会失败
  3. 运行时必须指定dll的路径，或者将dll文件复制到系统dll依赖类路径下
